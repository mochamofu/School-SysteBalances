# 積立金会計 入力アシスタント

実物の「令和○年度生積立金」（学校徴収金個人別管理票）への**入力作業を一括で楽にする**パッケージです。
マスターファイルそのものには一切手を加えず、隣に置いた「アシスタントブック」から安全に書き込みます。

## 解決する痛み

| これまで | これから |
|---|---|
| クラス替えのたび、掲示用名簿を見ながら320人分の組・番号を1行ずつ手で修正 | 名簿を貼り付けて2クリック。氏名で自動照合し、同姓同名だけ確認すればよい |
| 支出のたび、同じ金額を315人分入力し、転退学・給付型・欠席の例外を1人ずつ直す | 件名と一人あたり金額を入れ、例外の生徒だけ表に書いて1クリック |
| 口座振替のたび、振替できなかった生徒を目で探しながら入金額を入力 | 未納者の精算番号だけ書けば、残り全員に自動で金額が入り、未納の印も自動 |
| 支出承認書・収入承認書に同じ情報をもう一度手書き | 一括入力と同時に承認書シートが自動で埋まる（印刷するだけ） |
| 年度末、決算書のために電卓で項目ごとの合計を出して直打ち | 1クリックで全項目の 人数・一人あたり・執行総額 が一覧になる |
| 精算書は紙に一括印刷のみ | 生徒ごとのPDF（組-番号_氏名.pdf）で一括保存も可能 |

## フォルダ構成

```
assistant/
├── src/
│   ├── build_assistant.py     … アシスタントブック(xlsx)を生成
│   └── make_practice_files.py … 練習用ダミーファイル(架空の氏名)を生成
├── vba/
│   ├── A00_インストール.bas    … 最初に1回実行（メニューにボタンを並べる）
│   ├── A01_共通.bas           … マスターを開く・自動バックアップ・構造チェック
│   ├── A02_名簿取込.bas       … 掲示用名簿の解析・照合・クラス替え反映・新入生登録
│   ├── A03_支出一括入力.bas    … 支出の一括入力＋支出承認書の自動作成
│   ├── A04_収入一括入力.bas    … 収入(口座振替)の一括入力＋収入承認書の自動作成
│   ├── A05_決算集計.bas       … 決算書づくり用の項目別集計
│   ├── A06_整合性チェック.bas  … 収支の突き合わせ・未納者一覧などの点検
│   ├── A07_精算書一括PDF.bas   … 精算書の生徒別PDF一括保存
│   └── A08_精算書一括印刷.bas  … マスター内蔵の1999年製印刷マクロの安全な改善版
└── output/ (生成物・git管理外)
    ├── 積立金入力アシスタント.xlsx
    ├── 練習用_令和X年度生積立金.xlsx
    ├── 練習用_掲示用名簿.xlsx
    └── 練習用_空のマスター.xlsx
```

## 導入手順（学校PC・Microsoft 365 Excel）

1. `python assistant/src/build_assistant.py` で生成した `積立金入力アシスタント.xlsx` と、`vba/` の .bas 9本を学校PCへコピー
2. アシスタントブックを開き、Alt+F11 → ファイル → ファイルのインポート で A00〜A08 を全部インポート
3. 「Excelマクロ有効ブック(*.xlsm)」として保存し直す
4. Alt+F8 → `初期設定` を実行 → メニューシートにボタンが並ぶ
5. 「設定」シートに マスターファイルの場所（C3）と年度（C5）を入力

以後はメニューのボタンだけで操作できます。

## 安全装置

- **書き込み前に必ず自動バックアップ**（タイムスタンプ付きコピーを「バックアップ」フォルダへ）
- **構造チェック**：マスターの「データ」シートの形が想定と違うファイルには書き込まない
- **上書き確認**：すでに金額が入っている列に書こうとすると人数を示して確認
- マスターの数式列（H・I・BA〜BC・FC・FD）と精算番号・氏名には書き込まない設計
- 行や列の挿入・削除・並べ替えは一切行わない（精算書の数式が壊れないことを最優先）

## 検証について

このリポジトリでは、実物ファイル（匿名化済み）のコピーに対して VBA と同一ロジックの
Python シミュレーションを実行し、LibreOffice で再計算して以下を確認済みです。

- 掲示用名簿（8クラス×約40名のブロック配置）の自動解析
- 支出一括入力（例外：対象外0円・返金マイナス・個別金額）後の 支出合計・残金・合計行・精算書の正しい反応
- 収入一括入力（未納者スキップ）後の 収入合計・未納判定の正しい反応
- クラス替え80名の照合・反映のフルサイクル（不一致0件）
- 決算集計が実データの項目別合計と一致すること

※VBA本体はWindows + Microsoft 365 Excel上で動く前提です。導入時は必ず
`練習用_令和X年度生積立金.xlsx`（架空の氏名のダミー）で一連の操作を試してから
本物のマスターに使ってください。
