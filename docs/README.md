## 実機（Microsoft 365 Excel）で問題なく動くか

この開発・検証作業はLinux上のLibreOffice（24.2系）で行いました。LibreOffice 24.2にはXLOOKUP関数が実装されておらず（24.8以降で対応）、また日本語ロケール設定もないため、以下の2点は**この検証環境だけの制約**であり、Microsoft 365 Excel（日本語版）では問題なく動作する想定です。

- `XLOOKUP` を使った数式（③master_マスターブックの支出計算、④口座データバリデーションの氏名参照など）
- `LENB`/`LEN` を使った半角・全角カナの自動判定

逆に、この検証で実際に見つかり修正した3件のバグ（返金の相殺漏れ／決算書合計のズレ／口座データバリデーションの氏名未マッチ）は、関数のバージョン差とは無関係な**ロジック自体の誤り**だったため、Microsoft 365上でも同じ修正が必要でした（修正済み・コミット済みです）。

CoPilotプロンプトガイド（②）については、実際にプロンプトが動作するかは **Microsoft 365 Copilot のライセンス**が付与されたアカウントでのみ確認できます。

## 実機で試すときの手順（概要）

1. このリポジトリの `output/` フォルダ（`python src/generate_all.py` 実行後に生成される）の4つのExcelファイルと `vba/` フォルダをUSBメモリ等でMicrosoft 365 Excelが入った実機にコピーする
2. `動作確認_2台PCシミュレーション手順書.pdf` の「1. 事前準備」〜「6. VBAマクロの導入・動作確認」を順番に実施する
3. 各ステップで迷ったら `animations/` の該当するGIFを見ながら操作する
4. 「9. 最終チェックリスト」が全て埋まったら、学校PCへの本番導入へ進められます
