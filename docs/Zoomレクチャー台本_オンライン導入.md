## 当日の進行（60分）

### 0. 開始前（自分側の準備）
- 手元で同じ練習データを開いておく（詰まった学校の画面を再現するため）
- 学校が「画面共有」できるかが今日の生命線。開始5分で必ず確認

### 1. 0〜10分: 接続確認＋手順2の検収
- あいさつ→「今日はすべて練習用の架空データで行います。本物のファイルには一切触りません」を最初に宣言（安心して操作してもらうため）
- **学校側に画面共有してもらう**（こちらの画面を見せるのではなく、向こうに操作させるのが原則。見て覚えるのではなく手で覚えてもらう）
- 積立金入力アシスタント.xlsm を開いてもらい、警告バーの有無を確認
  - 赤バー「ブロックされました」→ 手順書5の1つ目（プロパティ→許可する）を口頭誘導
  - 黄バー「マクロが無効にされました」→「コンテンツの有効化」を押すだけ
  - トラストセンターに入れない → 管理者不在ならこの場は「コンテンツの有効化」運用で進め、後日管理者向けの1枚（手順書の手順2）をメールで送る

### 2. 10〜20分: 設定シート
- 「設定」シートC3に 練習用_令和X年度生積立金.xlsx のフルパスを入れてもらう
  - パスの取り方を教える: ファイルを Shift+右クリック →「パスのコピー」
- C5に年度、C7に 練習用_口座マスター.xlsx のパス
- ここで一言:「この黄色いセル2つを本物のファイルに書き換えるだけで、今日の練習がそのまま本番になります」

### 3. 20〜45分: 練習データで一巡（学校側が操作）
毎回、**先に「何が起きるはずか」を言ってから**押してもらう。

| 操作 | 期待結果（口頭で予告する数字） |
|---|---|
| 名簿貼付シートに 練習用_掲示用名簿 を貼る → ① | 4クラス80名を検出、全員一致 |
| ④ 支出一括入力（例: 校外学習 3,500円・全員） | 対象80名・合計280,000円、支出承認書が自動で埋まる |
| 振替結果取込シートに 練習用_振替結果 を貼る → ⑪ | 読取80件／振替済78／未納2（精算番号7・44）。未納者表に自動転記 |
| ⑤ 収入一括入力（例: 76,000円） | 入金78名・未納2名、マスターのH列に未納印 |
| 練習用マスターを開いて見せる | 「バックアップ」フォルダに控えが自動で増えていることを見せる（安全装置の実感） |

- 途中で1回、**わざと間違えさせる**（例: 精算番号400を入れて④）→ エラーで止まることを見せ、「間違えても壊れない」を体感してもらう

### 4. 45〜55分: 年間の運用と年度替わり
- ⑫ 年間予定表→フォーム転送（毎回件名・金額を打たない運用）
- ⑬→○×→⑭→⑮ を**こちらの画面共有で**デモ（操作は年度末なので今日は見るだけでよい）
- 「今年これ要らない→×、新しい項目→予定表に1行」だけ覚えて帰ってもらう

### 5. 55〜60分: クロージング
- 並走期間の約束: 「今月の振替結果が届いたら、マスターの**コピー**で⑪〜⑤をやってみて、手入力の結果と見比べてください。数字が合ったら次回Zoom（15分）で本番切替です」
- 連絡方法の確認: エラー画面はスマホで撮ってメール添付
- 次回日程（並走レビュー15分）をその場で決める ← 日付を決めずに終わらない

## リモート特有のトラブルと逃げ道

| 事象 | その場の対処 |
|---|---|
| 学校が画面共有できない | スマホでPC画面を撮ってカメラに映してもらう／こちらの画面共有に切替え、同じ操作を並走 |
| マクロがどうしても許可できない | 管理者向け依頼文（手順書の手順2の抜粋）をその場でメール。レクチャーは こちらの画面共有デモに切替えて完走し、設定完了後に15分の再接続枠を設定 |
| Excelが古い（2013以前） | ④⑤は動く可能性が高いが未検証と伝え、PDF系（⑨）は非対応の旨案内。更新を推奨 |
| USB使用禁止の学校 | 学校指定のファイル受け渡し方法（校内共有フォルダ等）に合わせ、ZIPを事前送付。手順は同じ |
| 通信が切れた | 電話に切替（発送メールに携帯番号を書いておく） |

## 1人で回すための時間割の目安（1校あたり）

- 発送準備（USB複製・検査・メール）: 45分
- Zoomレクチャー: 60分＋前後15分
- 並走レビューZoom: 15分×2回
- 本番切替立ち会い: 30分
- **合計 約3.5時間/校** → 週3校ペースで無理なく回せる（月10〜12校）
