# 全国展開の運用設計メモ（1人・オンライン導入体制）

前提: 都内3〜5校への訪問導入で型ができた後、訪問なし（郵送USB＋Zoom）で
全国の学校に導入する。運用者は1人。

## 導入フローの全体像

```
契約（LP→フォーム→オンライン商談）
  ↓
USB2本を作って郵送（＋発送メールで手順書PDFとZoom日程を送付）
  ↓
学校側がZoom前日までに 手順1〜3（コピー＋マクロ許可）を自力で実施
  ↓
Zoomレクチャー60分（練習データで一巡・学校側が操作）
  ↓
並走期間1〜2か月（マスターのコピーで実データを再現→手入力と突合）
  ↓
本番切替（Zoom 30分立ち会い）→ 月次サポートへ移行
```

## なぜ「郵送USB」がオンライン導入に向いているか

1. **Mark of the Web が付かない。** メール添付やダウンロードで渡した
   .xlsm はWindowsに「インターネット由来」の印が付き、マクロが強制
   ブロックされる（Microsoftの既定動作）。USBからのコピーはこの印が
   付かないため、リモートで一番つまずく関門を物理的に回避できる。
2. 学校のメール添付容量・ダウンロード制限に左右されない。
3. 「モノが届く」こと自体が、月額サービスの実体感につながる。

リスクと対策:
- **USB禁止の学校がある** → 発送前メールの事前チェック項目に入れた。
  禁止校には学校指定の受け渡し方法（校内共有等）でZIPを渡す（手順は同一）。
- **紛失** → USBには練習用の架空データと手順書しか入れない。
  実在の生徒・口座データは絶対に入れない（キット生成スクリプトの
  出荷前チェックリストにも明記）。紛失=個人情報事故にならない設計。

## 出荷用マスターUSBの作り方（Windowsで1回だけ）

学校側にVBAのインポートをさせない。これがリモート導入の成否を分ける。

1. 積立金入力アシスタント.xlsx に A00〜A11.bas をインポート
   → Alt+F8「初期設定」→ ボタン①〜⑮を確認 → **.xlsm で保存**
2. これが「出荷用マスターxlsm」。以後の出荷は
   `make_school_usb_kit.py` の生成物の .xlsx をこの .xlsm に
   差し替えて USB にコピーするだけ（1本あたり数分）。
3. 出荷前チェックリスト（生成物に同梱・手元用）で毎回検査:
   xlsm差し替え／作業メモ削除／動作検査①④⑤⑪／実データ不在確認／ラベル貼付。

バージョン管理: xlsm のファイル名は変えず、「設定」シートに
版数セルを設ける運用も可（改修版の配布は修正 .bas をメール →
学校でインポート…はさせない。**改修時も xlsm ごと差し替え**を郵送
またはZoom画面共有しながらの上書きコピーで行う）。

## 1人で回せる規模の試算

| 工程 | 時間/校 |
|---|---|
| 商談（オンライン30分＋見積・契約事務） | 1.0h |
| USB複製・出荷検査・発送 | 0.75h |
| Zoomレクチャー（前後込み） | 1.5h |
| 並走レビュー 15分×2 | 0.5h |
| 本番切替立ち会い | 0.5h |
| **導入1校あたり合計** | **約4.25h** |

- 週の稼働の半分を導入に充てるとして週20h → **週4校・月16校が上限目安**。
  安全側で **月10校** を公称キャパにする。
- 導入後の定常サポートは「エラー時のみ＋年度更新1回」なので、
  100校規模まで1人で保持可能（年度更新が3〜4月に集中する点だけ注意。
  ⑬⑭⑮で作業自体は学校が自走できるため、立ち会いは希望校のみに絞る）。

## 価格との整合

- スタンダード（月19,800円）の「導入支援1回」は **オンライン導入を標準** とする
  （LPの文言を「導入支援（オンライン）」に更新するとよい）。
- 訪問導入はプレミアム（34,800円）の特典、または出張実費＋日当のオプションに切り分け。
- 遠方校が増えるほど原価はUSB2本＋送料（1,500円程度）で一定 → 全国展開しても
  限界費用がほぼ増えない構造。

## 3〜5校導入後にやること（全国展開の引き金）

1. 導入事例（校名なしでも可）をLPに掲載、料金改定を検討（料金設計メモ参照）
2. Zoomレクチャーを1回録画し、「予習動画」として発送メールに添付
   （当日の60分を45分に短縮できる）
3. 手順書・台本の実戦フィードバック反映（つまずいた箇所を手順書5に追記）
4. 問い合わせ→商談→発送の管理が回らなくなる前に、簡単な顧客台帳を作る
   （学校名／契約プラン／導入日／年度更新日／Excelバージョン／連絡窓口）
