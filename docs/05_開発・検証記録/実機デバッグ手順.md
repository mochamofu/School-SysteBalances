# 実機デバッグ手順（Windows + Excel での最終確認）

対象: 製品パッケージ `積立金入力アシスタント_v1.0.0.zip` を、出荷できる状態か
自分の手で確かめるための手順。所要は通しで約90分（Phase 1〜4）。

前提の共有: VBAロジックはPythonミラー＋実ファイル検証で確認済みだが、
**Excel実機でのVBA実行はまだ一度も行われていない**。ここで初めて動く。
だからこのデバッグは「形式的な確認」ではなく本物の最終検証。

---

## Phase 0: 出荷用xlsmを作る（30分・一度きり）

1. パッケージZIPをWindows PCに展開し、`01_本体` をデスクトップへコピー
2. `積立金入力アシスタント.xlsx` を開く → Alt+F11 → ファイル → ファイルのインポート
   → `VBAモジュール予備` の A00〜A11 を12本すべてインポート
3. Alt+F8 → `初期設定` を実行 → メニューにボタン①〜⑮が並ぶことを確認
4. 「名前を付けて保存」→ ファイルの種類 **Excelマクロ有効ブック(.xlsm)** で保存
5. これが**出荷用マスターxlsm**。以後の全納品はこのファイルの複製でよい
   （インポート作業はもう二度と要らない）

つまずきポイント: インポート後にコンパイルエラーが出たら、VBE メニューの
デバッグ → VBAProject のコンパイル を押し、赤くなった行をスクショして送ること。
文字化けしている場合は .bas の文字コード問題なので、その旨だけ伝えれば対処する。

## Phase 1〜4: チェックシートを上から順に（60分）

`05_動作確認/動作確認チェックシート.pdf` を印刷し、A→B→C→Dの順に実行して
○×を付ける。期待値はすべてシートに書いてある。要点だけ再掲:

- **B（練習80名）**: ①80名一致 → ④280,000円 → ⑪読取80/済78/未納2（7・44）→ ⑤入金78
- **C（検証320名）**: ①320名一致、⑪読取321/済315/未納5（7・44・159・241・312）/不明1
- **D（わざと壊す）**: 精算番号400→停止、C3空→案内、上書き→確認、他ファイル→拒否

Bの途中で失敗しても、練習用マスターはパッケージからコピーし直せば何度でも
最初からやり直せる（本物のデータは一切使っていないので壊すものがない）。

## エラーが出たときの報告テンプレ（これを送れば最短で直る）

```
ボタン: ⑪振替結果を照合
メッセージ: 実行時エラー '9' インデックスが有効範囲にありません
（エラーダイアログで「デバッグ」を押し、黄色くなった行のスクショを添付）
直前の操作: 検証用_振替結果.xlsx の321行を貼り付けた直後
データ: 検証用320名
Excel: Microsoft 365 (バージョン2506)
```

「デバッグ」ボタン→黄色い行のスクショが一番効く。モジュール名と行が
特定できるので、ピンポイントで修正版を返せる。

## 修正版の適用方法（修正.basを受け取ったら）

1. Alt+F11 → 左のツリーで該当モジュール（例: A09_振替結果取込）を右クリック
   → 「A09_振替結果取込の解放」→ エクスポートしない
2. ファイル → ファイルのインポート → 受け取った新しい .bas を選択
3. 上書き保存 → チェックシートの該当項目だけ再実行（全部やり直す必要はない）

## よくある環境系エラー（製品のバグではないもの）

| 症状 | 原因と対処 |
|---|---|
| 実行時エラー1004「ファイルが見つかりません」 | 設定C3/C7のパスの打ち間違い。エクスプローラーで Shift+右クリック→「パスのコピー」で貼り直す |
| 開いた瞬間に赤い「ブロックされました」バー | Mark of the Web。ファイル右クリック→プロパティ→「許可する」 |
| ボタンを押しても無反応 | マクロ無効のまま。黄色バーの「コンテンツの有効化」または信頼できる場所の設定 |
| ⑨PDF保存でエラー | PDF仮想プリンタ関連。1名分だけ再実行して再現するか確認してから報告 |

## 全部○になったら

- 出荷用xlsmを `01_本体` に戻し、`★出荷前に差し替え.txt` と
  `VBAモジュール予備` を削除 → ZIPを作り直せば**納品可能な完成品**
- 検証済みの印として、チェックシートに日付とExcelバージョンを書いて保管
  （学校から動作環境を聞かれたときの根拠になる）
