# 支出の部の確認結果と、業者別集計・引き継ぎ機能（⑬⑭⑮）

実施日: 2026-07-06
対象: 実物マスター「令和○年度生積立金」の匿名化コピー（支出の部 BE〜FB列）

## 1. 確認したこと（ご質問への回答）

### 支出の部は業者名の列で作られているか → はい

マスターのデータシート6行目（項目名の行）を全100枠確認しました。
使用中は29枠で、すべて「ベネッセテスト代金」「クラッシィ追加」「漢字検定」
「田中金華堂」のような業者・案件名で列が作られています。
未使用の71枠は「支出　28」のような仮見出しのままでした。

### 各生徒に毎月の請求額がそのまま入っているか → はい

29項目それぞれについて、生徒321行分の金額を1列ずつ独立に再集計し、
一覧の「対象人数・執行総額」と**全項目で1円の差もなく一致**することを
確認しました（不一致 0件）。ほとんどの項目は全員同額
（例：クラッシィ 314名×3,960円）ですが、漢字検定のように
2,000／2,500／3,500円の3種類が混在する項目、給付型奨学金のように
マイナス（返金・調整）が入る項目もあり、いずれも生徒ごとの実額が
そのまま入っています。

### アシスタントは数字を加工せずに転記するか → はい

- ④支出一括入力は、入力した一人あたり金額（と例外表の個別金額）を
  **そのままの値**でマスターの該当列に書き込みます。端数処理・按分などの
  加工は一切ありません（意地悪テストT13で、書き込み先が想定列のみで
  あることも確認済み）。
- 今回追加した⑬⑭⑮は**マスターを読むだけ**で、1円も書き換えません。

## 2. 見つかった「見えにくさ」と新機能

同じ業者が複数の列に分かれているため、「この業者に年間いくら」が
ひと目では分かりませんでした。

| 業者 | 分かれていた列 | 年間合計 |
|---|---|---|
| 田中金華堂 | 5列（No.14,15,16,19,23） | 586,112円 |
| クラスTシャツ | 2列（No.11,18） | 376,800円 |
| 教材費 小笠原 | 2列（No.2,3） | 45,072円 |

これを解決するのが新モジュール **A11_支出項目管理**（ボタン⑬⑭⑮）です。

### ⑬ 支出項目を読み込む
マスターの支出100枠を「支出項目一覧」シートに一覧化
（No・件名・日付・対象人数・一人あたり・執行総額）。
業者名（C列）・○×（H列）・メモ（I列）は手で編集でき、
読み直しても消えません（支出Noで引き当てて保持）。

### ⑭ 業者別に集計する
C列の業者名が同じ行をまとめ、業者ごとの
**項目数・年間合計・月別（4月〜3月）**を右側に出力。
同じ業者名にそろえるだけで、5列に分かれた田中金華堂も1行に合算されます。

### ⑮ 来年度の予定へ引き継ぐ
H列に○を付けた項目だけを「年間予定表」に自動追加します。

- **「今年これ要らない」** → ×にする（または予定表から行を消す）だけ
- **「今年これ追加したい」** → 年間予定表に直接1行書き足すだけ
- 実行時は ⑫予定を入力フォームへ転送 → ④一括入力 の2クリック

3年間で項目が入れ替わっても、○×の付け替えと行の書き足しだけで
毎年のファイル作りができます。
