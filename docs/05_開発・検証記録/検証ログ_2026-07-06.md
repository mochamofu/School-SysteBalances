# 動作テスト・検証ログ（意地悪テスト編）

実施日: 2026-07-06
対象: 積立金入力アシスタント VBA一式（A02/A03/A04/A09）＋生成物・キット

## この検証で何をやったか（ひとことで）

ハッピーパス（正常データを入れて動く）はすでに確認済みだったので、今回は
**わざと変なデータ・限界値・想定外の使い方をぶつける「意地悪テスト」**を
15項目行いました。VBA本体はこのサンドボックスで直接実行できないため、
VBAと1行1行同じ処理をするPythonの「ミラー関数」を書いて、実物ファイルの
匿名化コピーや検証用データに対して流し込む方式で検証しています。

結果: **15項目中7項目で問題が見つかり、すべて修正して再テスト全合格**にしました。

## 見つかった問題と修正（7件）

### ① 全角数字の口座番号が照合できない（T1）
- **やった検証**: 口座記号・番号に全角数字「１００００」を入れて⑪照合を通した。
- **起きたこと**: 照合キーが空になり、その生徒が全員「不明口座」に落ちる。
  学校の入力では全角数字が混ざることが普通にあるため実害あり。
- **原因**: `数字だけ` 関数が半角の 0〜9 しか拾っていなかった。
- **修正**: 関数の先頭で `StrConv(..., vbNarrow)` を入れ、全角→半角に統一してから
  数字を拾うようにした（A09_振替結果取込.bas）。

### ② 兄弟の共有口座で片方が振替済み扱いになる（T3）
- **やった検証**: 口座マスターに同じ口座を持つ2生徒（兄弟の共有口座）を登録し、
  その口座が資金不足だった振替結果を照合した。
- **起きたこと**: 辞書に最初の1人しか登録されず、2人目は照合対象から漏れて
  「払った扱い」になる（未納の見落とし＝一番危ない）。
- **修正**: 口座マスター読込時に「同じ口座が複数生徒にある」ことを検出し、
  その口座は「口座重複（要確認）」として**人間に返す**ようにした。
  機械が勝手にどちらか片方に付けない安全設計（A09）。

### ③ 振替結果に同じ口座が2回出ると二重計上（T6）
- **やった検証**: 銀行の結果票に同一口座を2行（1回済＋1回不足）入れて照合した。
- **起きたこと**: 同じ生徒が「振替済」と「未納」の両方でカウントされ、集計がズレる。
- **修正**: 照合済みの口座を記録し、2回目以降は「重複行（要確認）」として
  カウントから外し人間に返すようにした（A09）。

### ④ フルスケール321件を貼ると末尾が黙って消える（T4）★重要
- **やった検証**: 検証用の全校分321行（未納者は精算番号312も含む）を
  振替結果取込シートに貼り付ける想定で件数を確認した。
- **起きたこと**: 貼り付け枠が**300行しかなく**、末尾21行が黙って無視される。
  未納者（312番）が枠外に落ちて処理されない致命的な取りこぼし。
- **修正**: 貼付枠・未納者表・例外表の上限を **300行→1000行**に拡張。
  VBA側の定数（取込_終了行・未納_終了行・例外_終了行）と
  シート生成側（build_assistant.py）の両方を揃えて、321件が全部読めるようにした。

### ⑤ 範囲外の精算番号が黙って無視される（T10）
- **やった検証**: 支出の例外表・収入の未納者表に、存在しない精算番号「400」を入れた。
- **起きたこと**: 範囲チェックが無く黙って無視。入力ミスに気づけない。
- **修正**: 精算番号が 1〜321（在籍上限）の外なら、その場でエラー表示して
  実行を止めるようにした（A03・A04）。

### ⑥ 10人未満の小規模クラスが名簿解析で拾えない（T9）
- **やった検証**: 6人だけの特別支援学級のような小規模クラスの掲示名簿を解析した。
- **起きたこと**: 「10人以上ないと番号列・氏名列と認めない」閾値のせいで、
  そのクラスのブロックがまるごと検出されず0人になる。
- **修正**: 検出閾値を 10→3 に下げ、少人数クラスも拾えるようにした（A02）。

### ⑦ 結果コードの表記ゆれ（T7）※テスト側の誤検出だった
- **やった検証**: 振替結果を 0 / "00" / 数値の0 / "済" / "OK" / 空欄 など11通りで判定させた。
- **起きたこと**: 検証プログラム（Pythonミラー）側が、**数値の0** を空欄と誤判定していた。
- **切り分け**: VBA本体は `CStr(0)="0"` で正しく「振替済」と判定する。
  問題は検証プログラムの `str(v or '')` が Python の falsy 評価で 0 を空文字にしていたこと。
- **対応**: VBA本体は正常のため修正不要。**テスト側のミラー関数を修正**して再確認した。
  （テストの道具のバグを製品のバグと取り違えないよう、必ず切り分けた）
