## 学校のパソコンへの転送方法

1. `output/` フォルダ内の4つの `.xlsx` ファイルをUSBメモリにコピー
2. 学校のパソコンの任意のフォルダ（例: `デスクトップ\学次会計ツール\`）に貼り付け
3. VBAマクロを追加する（下記「VBAの追加方法」参照）

## VBAマクロの追加方法

> **VBAを知らない方へ**: マクロとはExcelの自動処理プログラムです。一度設定すれば「全生徒の精算書を印刷」がボタン1つでできるようになります。

### ③ マスターブックへのマクロ追加
1. `master_マスターブック.xlsx` を Excel で開く
2. `Alt + F11` を押す（VBAエディタが開く）
3. メニュー「ファイル」→「インポート」→ `vba/MasterBook_Macros.bas` を選択
4. `Alt + F11` でExcelに戻る
5. 「名前を付けて保存」→ファイルの種類を **「マクロ有効ブック (.xlsm)」** に変更して保存

### ① 番号紐付けテンプレートへのマクロ追加
同じ手順で `vba/ClassLinking_Macros.bas` をインポートしてください。

## 各ツールの使い方

### ① 番号紐付けテンプレート（クラス替え時・毎年3〜4月）

```
先生から新クラス名簿を受領
  ↓
「新年度名簿入力」シート（橙タブ）に名簿を貼り付け
  ↓
「照合結果」シート（赤タブ）で全員「一致」を確認
  ↓
「生徒マスター」シートの年度列にクラス情報を転記
```

**CoPilotへの質問例**:
```
このシートで一致していない生徒を一覧にして
```

---

### ② CoPilotプロンプトガイド（毎日）

黄色のプロンプト文をコピーしてOutlookのCoPilot（虹色丸マーク）に貼り付けるだけ。

**よく使うプロンプト**:
```
昨日届いたメールで、本庁からの通知や期限付きの依頼があれば教えてください
```

---

### ③ マスターブック（日常の収支管理）

**入力するシートは3つだけ**

| シート | タイミング |
|--------|-----------|
| 生徒マスター（緑） | 年度初め・転退学時 |
| 月次徴収（青） | 口座引落後 |
| 支出記録（赤） | 支出発生のたびに |

**精算書の出し方**:
1. 「精算書(出力)」シートを開く
2. **黄色のB3セル**に生徒番号を入力 → 自動表示
3. VBAマクロ「一括印刷」で全員分を印刷

**CoPilotへの質問例**:
```
このExcelの支出記録シートで今年度の支出を教科別に集計してください
```

---

### ④ 口座データバリデーション（毎年4月）

```
紙の申込書（約300件）から水色セルに入力
  ↓
「バリデーション結果」シートのK列「総合判定」を確認
  ↓
「要確認」の行だけ修正（推定10件以下）
  ↓
「銀行提出用」シートの16桁コードを銀行に提出
```

## よくある質問

**Q. 「マクロを有効にしますか？」と聞かれた**  
→「コンテンツの有効化」をクリックしてください。

**Q. 生徒番号の先頭の0が消えてしまう**  
→ 列を選択→右クリック→セルの書式設定→「文字列」に変更して入力し直してください。

**Q. 数式が `#REF!` になっている**  
→ 出力シートは触らないでください。入力シートのデータを確認してください。

**Q. 精算書にゼロが表示される**  
→ VBAマクロ「精算書_行非表示」を実行してください。

**Q. CoPilotのアイコンが見つからない**  
→ IT管理者に「CoPilot for Microsoft 365を有効にしてほしい」と依頼してください。

## 実機（Microsoft 365 Excel）で問題なく動くか

この開発・検証作業はLinux上のLibreOffice（24.2系）で行いました。LibreOffice 24.2にはXLOOKUP関数が実装されておらず（24.8以降で対応）、また日本語ロケール設定もないため、以下の2点は**この検証環境だけの制約**であり、Microsoft 365 Excel（日本語版）では問題なく動作する想定です。

- `XLOOKUP` を使った数式（③master_マスターブックの支出計算、④口座データバリデーションの氏名参照など）
- `LENB`/`LEN` を使った半角・全角カナの自動判定

逆に、この検証で実際に見つかり修正した3件のバグ（返金の相殺漏れ／決算書合計のズレ／口座データバリデーションの氏名未マッチ）は、関数のバージョン差とは無関係な**ロジック自体の誤り**だったため、Microsoft 365上でも同じ修正が必要でした（修正済み・コミット済みです）。

CoPilotプロンプトガイド（②）については、実際にプロンプトが動作するかは **Microsoft 365 Copilot のライセンス**が付与されたアカウントでのみ確認できます。

## 実機で試すときの手順（概要）

1. このリポジトリの `output/` フォルダ（`python src/generate_all.py` 実行後に生成される）の4つのExcelファイルと `vba/` フォルダをUSBメモリ等でMicrosoft 365 Excelが入った実機にコピーする
2. `動作確認_2台PCシミュレーション手順書.pdf` の「1. 事前準備」〜「6. VBAマクロの導入・動作確認」を順番に実施する

4. 「9. 最終チェックリスト」が全て埋まったら、学校PCへの本番導入へ進められます
