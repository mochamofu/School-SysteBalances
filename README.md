# 学次会計 業務改善ツール

私学高等学校の学次会計業務を効率化するExcelツール一式です。

---

## 含まれるツール

| # | ファイル名 | 目的 | 使う時期 |
|---|-----------|------|---------|
| ① | `番号紐付けテンプレート.xlsx` | クラス替え時の出席番号更新 | 毎年3〜4月 |
| ② | `CoPilotプロンプトガイド_学次会計用.xlsx` | 本庁通知・メールの検索ガイド | 毎日 |
| ③ | `master_マスターブック.xlsx` | 収支管理・精算書・決算書の自動生成 | 常時 |
| ④ | `口座データバリデーション.xlsx` | 口座申込書の入力ミスチェック | 毎年4月 |

---

## セットアップ（初回のみ）

### 必要なもの
- Python 3.9 以上
- Microsoft Excel 365 または Excel 2019以上（Windows推奨）

### インストール
```bash
pip install -r requirements.txt
```

### Excelファイルの生成
```bash
python src/generate_all.py
```
`output/` フォルダに4つのExcelファイルが生成されます。

---

## 学校のパソコンへの転送方法

1. `output/` フォルダ内の4つの `.xlsx` ファイルをUSBメモリにコピー
2. 学校のパソコンの任意のフォルダ（例: `デスクトップ\学次会計ツール\`）に貼り付け
3. VBAマクロを追加する（下記「VBAの追加方法」参照）

---

## VBAマクロの追加方法

> **VBAを知らない方へ**: マクロとはExcelの自動処理プログラムです。一度設定すれば「全生徒の精算書を印刷」がボタン1つでできるようになります。

### ③ マスターブックへのマクロ追加
1. `master_マスターブック.xlsx` を Excel で開く
2. `Alt + F11` を押す（VBAエディタが開く）
3. メニュー「ファイル」→「インポート」→ `vba/MasterBook_Macros.bas` を選択
4. `Alt + F11` でExcelに戻る
5. 「名前を付けて保存」→ファイルの種類を **「マクロ有効ブック (.xlsm)」** に変更して保存

### ① 番号紐付けテンプレートへのマクロ追加
同じ手順で `vba/ClassLinking_Macros.bas` をインポートしてください。

## 各ツールの使い方

### ① 番号紐付けテンプレート（クラス替え時・毎年3〜4月）

```
先生から新クラス名簿を受領
  ↓
「新年度名簿入力」シート（橙タブ）に名簿を貼り付け
  ↓
「照合結果」シート（赤タブ）で全員「一致」を確認
  ↓
「生徒マスター」シートの年度列にクラス情報を転記
```

**CoPilotへの質問例**:
```
このシートで一致していない生徒を一覧にして
```

---

### ② CoPilotプロンプトガイド（毎日）

黄色のプロンプト文をコピーしてOutlookのCoPilot（虹色丸マーク）に貼り付けるだけ。

**よく使うプロンプト**:
```
昨日届いたメールで、本庁からの通知や期限付きの依頼があれば教えてください
```

---

### ③ マスターブック（日常の収支管理）

**入力するシートは3つだけ**

| シート | タイミング |
|--------|-----------|
| 生徒マスター（緑） | 年度初め・転退学時 |
| 月次徴収（青） | 口座引落後 |
| 支出記録（赤） | 支出発生のたびに |

**精算書の出し方**:
1. 「精算書(出力)」シートを開く
2. **黄色のB3セル**に生徒番号を入力 → 自動表示
3. VBAマクロ「一括印刷」で全員分を印刷

**CoPilotへの質問例**:
```
このExcelの支出記録シートで今年度の支出を教科別に集計してください
```

---

### ④ 口座データバリデーション（毎年4月）

```
紙の申込書（約300件）から水色セルに入力
  ↓
「バリデーション結果」シートのK列「総合判定」を確認
  ↓
「要確認」の行だけ修正（推定10件以下）
  ↓
「銀行提出用」シートの16桁コードを銀行に提出
```

## CoPilot 連携ポイント

このツールはCoPilot for Microsoft 365（Work版）との連携を想定しています。  
各シートの「操作ガイド」シートにCoPilotへの質問例を記載しています。

| ツール | CoPilotが担う役割 |
|--------|-----------------|
| ①番号紐付け | 未マッチ・不一致行の原因分析、転入生か表記ゆれかの判定支援 |
| ②プロンプトガイド | 本庁通知の横断検索・期限タスクの自動抽出 |
| ③マスターブック | 支出データの集計・分析・異常値の発見 |
| ④口座バリデーション | エラー行の原因特定・修正方法の提案 |

## よくある質問

**Q. 「マクロを有効にしますか？」と聞かれた**  
→「コンテンツの有効化」をクリックしてください。

**Q. 生徒番号の先頭の0が消えてしまう**  
→ 列を選択→右クリック→セルの書式設定→「文字列」に変更して入力し直してください。

**Q. 数式が `#REF!` になっている**  
→ 出力シートは触らないでください。入力シートのデータを確認してください。

**Q. 精算書にゼロが表示される**  
→ VBAマクロ「精算書_行非表示」を実行してください。

**Q. CoPilotのアイコンが見つからない**  
→ IT管理者に「CoPilot for Microsoft 365を有効にしてほしい」と依頼してください。
